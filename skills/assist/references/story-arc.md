# Story arc — narrative shapes

Once you have an outline (outline-method.md), order it as a *story*, not a table of contents.
A reader follows tension → resolution; pick the arc that fits the goal and the audience.

## Pick by intent

| Goal | Arc | Shape |
|------|-----|-------|
| Persuade / pitch | **SCR (Situation–Complication–Resolution)** | "Here's where we are → here's the problem → here's the answer." The default for decks that ask for a decision or money. |
| Update / report | **Answer-first** | Lead with the verdict (on track / at risk), then the evidence, then what's next. Execs want the bottom line on slide one. |
| Teach / onboard | **Progressive** | Simple → complex, each step building on the last. Concept, then example, then edge cases. |
| Launch / sell | **PAS (Problem–Agitate–Solution)** | Name the pain → make it vivid → present the product as relief → proof → call to action. |
| Inspire / vision | **Before–After–Bridge** | The world today → the better world → the bridge you're offering to cross it. |

## Universal beats

Whatever the arc, most strong documents have:

- **A hook** — the first slide/section earns the next. A surprising number, a sharp question,
  the stakes. Never open with "Agenda".
- **A turn** — the moment of tension: the complication, the problem, the gap. Without it the
  piece is a list, not a story.
- **Proof** — evidence the audience can't wave away: data, a demo, a customer's words.
- **A single ask** — end on one clear next step. A deck that asks for three things gets none.

## Validation lenses (quick self-check before you hand back a draft)

- **Clarity** — could a stranger state the main message after one read?
- **Evidence** — is every claim backed, and every number grounded (wicked-brain)?
- **Flow** — does each section earn the next, or could you shuffle them with no loss? (If you
  could, the arc is weak.)
- **Audience fit** — right altitude for who's reading? An exec update isn't an engineering
  deep-dive.

If a draft fails a lens, fix the structure before the styling — surface polish can't rescue a
broken argument, and the user will feel the wrongness without being able to name it.
