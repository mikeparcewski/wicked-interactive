# Outline method — structure before HTML

Never start from a blank slide. Turn the source material (or the brief) into a **structured
outline** first, then render that outline to HTML. A first draft that's well-structured but
plain beats a pretty draft that rambles — the user refines surface later by clicking.

## The Pyramid Principle (Barbara Minto)

Lead with the answer; support it underneath. For every document and every section:

1. **Governing thought first.** State the single main message up front — the thing the reader
   should remember if they read nothing else. ("Q3 beat plan: revenue +18%, margin held.")
2. **Group supporting points** under it — 3 to 5, each a complete thought, mutually exclusive,
   collectively covering the claim (MECE). Not a list of facts; a logical argument.
3. **Recurse** — each supporting point is itself a mini-pyramid (its own lead + evidence).
4. **Order deliberately** — by importance, by time, or by structure; never arbitrarily.

Why it works for an exec audience: they can stop at any level and still have the gist. The
deeper they read, the more evidence they get — never the reverse.

## From material to outline

1. **Extract claims, not sentences.** Read every source; pull out the assertions and the
   numbers that back them. Drop prose you won't use.
2. **Find the one governing thought.** If you can't state it in a sentence, you don't have a
   document yet — you have notes. Ask the user if the brief is ambiguous.
3. **Cluster** the claims into 3–5 groups. Name each group as a *message* ("Costs are now
   predictable"), not a *topic* ("Costs").
4. **Sequence** the groups into an arc (see story-arc.md).
5. **One slide / one section per group**, with the group's message as the heading and its
   evidence as the body. A slide with two messages is two slides.

## Slide/section discipline

- **One idea per slide.** The heading is the takeaway, not a label. "Revenue grew 18%" beats
  "Revenue".
- **Assertion–evidence.** Heading asserts; body proves (a chart, a number, a short list).
- **Tight body.** 3–5 bullets max, or one short paragraph. If it's dense, split it.
- **Ground every number.** Use the figures the source supports; if wicked-brain has the
  canonical value, use that and say so. Never invent a statistic to fill a slot.

## Length

Honor the brief. With none: a deck is ~6–10 slides for an update, ~12–18 for a pitch; a
one-pager is one screen; a landing page is hero → value → proof → CTA. When in doubt, fewer,
sharper sections — the user adds more by asking.
